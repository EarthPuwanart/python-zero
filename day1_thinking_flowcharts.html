คุณสมบัติของ Algorithm ที่ดี

Algorithms

☕ ตัวอย่าง Algorithm: ชงกาแฟ

ขั้นตอน (Steps)

1. เริ่มต้น (START)
2. ต้มน้ำร้อน
3. ใส่ผงกาแฟลงในแก้ว
4. เทน้ำร้อนลงในแก้ว
5. เติมน้ำตาลหรือไม่? (Yes/No)
   5a. ถ้าใช่ (Yes) → เติมน้ำตาลและคนให้เข้ากัน
   5b. ถ้าไม่ (No) → ข้ามขั้นตอนนี้
6. เติมนมหรือไม่? (Yes/No)
   6a. ถ้าใช่ (Yes) → เติมนมและคนให้เข้ากัน
   6b. ถ้าไม่ (No) → ข้ามขั้นตอนนี้
7. เสิร์ฟกาแฟ
8. จบการทำงาน (END)

สังเกต

  • มีจุดเริ่มต้น (START) และจุดสิ้นสุด (END)
  • แต่ละขั้นตอนชัดเจน ทำตามได้
  • มี Decision (ตัดสินใจ) — ใส่น้ำตาล? ใส่นม?
  • ผลลัพธ์ขึ้นอยู่กับ Input ของผู้ใช้

💡 Algorithm ไม่ได้มีแค่ในคอมพิวเตอร์ — สูตรอาหาร คู่มือประกอบเฟอร์นิเจอร์ ก็เป็น Algorithm!

Algorithms

🎓 ตัวอย่าง Algorithm: คำนวณเกรด

ขั้นตอน (Steps)

1. รับคะแนนสอบ (score)
2. ถ้า score ≥ 80 → grade = "A"
3. ถ้า score ≥ 70 → grade = "B"
4. ถ้า score ≥ 60 → grade = "C"
5. ถ้า score ≥ 50 → grade = "D"
6. อื่นๆ → grade = "F"
7. แสดงผล grade

ทดสอบด้วย score = 75

1. รับค่า: score = 75
2. 75 ≥ 80 ? (เท็จ) เลื่อนไปข้อ 3
3. 75 ≥ 70 ? (จริง) → grade = "B"
7. แสดงผล: "B"

💡 โปรแกรมจะตรวจสอบเงื่อนไขตามลำดับจากบนลงล่าง เมื่อเจอเงื่อนไขที่เป็น "จริง" แล้ว จะไม่ตรวจสอบเงื่อนไขด้านล่างอีก

โครงสร้างแบบลำดับ (Sequential)

ทำงานเรียงลำดับจากบนลงล่าง ทีละขั้นตอน ไม่มีการข้ามหรือวนซ้ำ

คำนวณพื้นที่วงกลม

ขั้นตอน (Steps)

1. เริ่มต้น (START)
2. กำหนดให้รัศมี (radius) = 5
3. คำนวณพื้นที่ (area) = 3.14 × radius × radius
4. แสดงผลพื้นที่ (area)
5. จบการทำงาน (END)

💡 โครงสร้างแบบลำดับเป็นโครงสร้างที่เรียบง่ายที่สุด โปรแกรมส่วนใหญ่ทำงานโดยใช้โครงสร้างแบบลำดับเป็นพื้นฐาน

Flowcharts

โครงสร้างแบบลำดับ (Sequential)

ทำงานเรียงลำดับจากบนลงล่าง ทีละขั้นตอน ไม่มีการข้ามหรือวนซ้ำ

คำนวณราคาสุทธิ (รวม VAT 7%)

ขั้นตอน (Steps)

1. เริ่มต้น (START)
2. รับค่า ราคาสินค้า (price)
3. คำนวณ ราคาสุทธิ (total) = price × 1.07
4. แสดงผล ราคาสุทธิ (total)
5. จบการทำงาน (END)

💡 1.07 มาจาก 100% (ราคาเดิม) + 7% (VAT) เขียนแบบนี้เพื่อรวบขั้นตอนการคำนวณให้สั้นลง

Flowcharts

โครงสร้างแบบเลือก (Decision / Selection)

ตรวจสอบเงื่อนไข แล้วทำตาม Yes หรือ No

ตรวจสอบเลขคู่ / เลขคี่

ขั้นตอน (Steps)

1. เริ่มต้น (START)
2. รับค่า ตัวเลข (num)
3. ตรวจสอบว่า num หารด้วย 2 ลงตัวหรือไม่
   3a. ถ้าจริง → แสดงผล "Even" (เลขคู่)
   3b. ถ้าเท็จ → แสดงผล "Odd" (เลขคี่)
4. จบการทำงาน (END)

💡 เครื่องหมาย % (Modulo) ใช้หาเศษจากการหาร ถ้าหารด้วย 2 แล้วเศษเป็น 0 แสดงว่าเป็นเลขคู่เสมอ

Flowcharts

โครงสร้างแบบเลือก (Decision / Selection)

ตรวจสอบเงื่อนไข แล้วทำตาม Yes หรือ No

ตัดเกรดจากคะแนนสอบ

ขั้นตอน (Steps)

1. เริ่มต้น (START)
2. รับคะแนนสอบ (score)
3. ตรวจสอบ score ตามลำดับขั้น:
   3a. ≥ 80 → ได้ 'A', ข้ามไปข้อ 4
   3b. ≥ 70 → ได้ 'B', ข้ามไปข้อ 4
   3c. ≥ 60 → ได้ 'C', ข้ามไปข้อ 4
   3d. ≥ 50 → ได้ 'D', ข้ามไปข้อ 4
   3e. น้อยกว่า 50 → ได้ 'F', ข้ามไปข้อ 4
4. แสดงผลเกรด (g)
5. จบการทำงาน (END)

💡 โครงสร้างแบบนี้เรียกว่า Nested If ใช้ในการตัดสินใจที่มีหลายเงื่อนไขต่อเนื่องกัน

Flowcharts

โครงสร้างแบบวนซ้ำ (Loop / Repetition)

ทำซ้ำขั้นตอนเดิม จนกว่าเงื่อนไขจะเปลี่ยน

แสดงเลข 1 ถึง 5

ขั้นตอน (Steps)

1. เริ่มต้น (START)
2. กำหนดให้ i = 1
3. ตรวจสอบว่า i ≤ 5 ใช่หรือไม่
   3a. จริง → แสดงผลค่า i, ให้ i = i + 1, แล้ววนกลับไปข้อ 3
   3b. เท็จ → ไปทำข้อ 4
4. จบการทำงาน (END)

⚠️ ข้อควรระวัง! ถ้าเงื่อนไขไม่เปลี่ยนแปลง จะเกิด Infinite Loop

Flowcharts

โครงสร้างแบบวนซ้ำ (Loop / Repetition)

ทำซ้ำขั้นตอนเดิม จนกว่าเงื่อนไขจะเปลี่ยน

หาผลรวมเลข 1 ถึง 3

ขั้นตอน (Steps)

1. เริ่มต้น (START)
2. กำหนด sum = 0, i = 1
3. ตรวจสอบว่า i ≤ 3 ใช่หรือไม่
   3a. จริง → sum = sum + i,
       เพิ่มค่า i = i + 1 แล้ววนกลับข้อ 3
   3b. เท็จ → ออกจากการวนซ้ำ ไปข้อ 4
4. แสดงผลลัพธ์ ผลรวม (sum)
5. จบการทำงาน (END)

โจทย์ที่ 1: คำนวณพื้นที่สามเหลี่ยม

โจทย์

รับค่าฐาน (base) และความสูง (height) จากนั้นคำนวณพื้นที่สามเหลี่ยม และแสดงผลลัพธ์

💡 คำใบ้: พื้นที่ = 0.5 * ฐาน * สูง

Algorithm

1. รับค่า base, height
2. area = 0.5 * base * height
3. แสดงผล area

(ลองฝึกวาด Flowchart ด้วยตัวเองนะ!)

โจทย์ที่ 2: ตรวจสอบสิทธิ์เลือกตั้ง

โจทย์

💡 คำใบ้: ต้องใช้โครงสร้างแบบตัดสินใจ (Decision)

โจทย์ที่ 3: ตรวจสอบเลขคู่/คี่

โจทย์

รับค่าตัวเลข n ตรวจสอบว่าเป็น 'เลขคู่' (Even) หรือ 'เลขคี่' (Odd)

💡 คำใบ้: เลขคู่ คือเลขที่หารด้วย 2 แล้วเหลือเศษ 0 (n % 2 = 0)

Algorithm

1. รับค่า n
2. ถ้า n % 2 = 0
   2a. จริง → แสดงผล "Even"
   2b. เท็จ → แสดงผล "Odd"

(ลองฝึกวาด Flowchart ด้วยตัวเองนะ!)

โจทย์ที่ 4: นับถอยหลัง (Countdown)

โจทย์

ตั้งค่าเริ่มต้นนับถอยหลังตั้งแต่ 3 เมื่อนับจนถึง 0 ให้แสดงคำว่า 'GO!'

💡 คำใบ้: ใช้ตัวแปร count=3 แล้วลดค่าลงทีละ 1 ใน Loop

Algorithm

1. count = 3
2. ถ้า count > 0
   2a. จริง → แสดงผล count, count = count - 1, ย้อนไป 2.
   2b. เท็จ → แสดงผล "GO!"

(ลองฝึกวาด Flowchart ด้วยตัวเองนะ!)

โจทย์ที่ 5: ระบบล็อกอิน
